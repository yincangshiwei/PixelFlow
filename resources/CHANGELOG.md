# PixelFlow 版本日志

---

## 2026-05-08

### 打包配置 — 修复 exe 运行时 `No module named 'http'` 错误

- 从 PyInstaller `excludes` 列表中移除 `http` 模块
- 根因：`python-pptx` 内部依赖 `http.client` 模块处理 OPC/XML 打包，原配置将其排除导致图片排版导出功能完全无法使用

---

## 2026-05-06

### 后台日志 — 新增按功能落盘与异常追踪

- 新增 `logs/` 运行日志目录，每个功能独立保存 `current.log` 实时日志和 `history/YYYY-MM-DD.log` 历史日志
- 后台日志窗口的内容会同步写入当前功能日志，清空日志仅清空实时日志，历史日志持续保留
- 未捕获异常和后台线程详细错误会写入日志文件，便于排查闪退、卡住和批量导出失败问题

### 图片排版导出 — 增强批量导出稳定性与错误记录

- 图片排版导出失败时记录完整异常堆栈到功能日志，日志窗口保持简洁错误提示
- 优化 Excel 读取和 PDF 压缩图片绘制的资源释放，降低连续多次导出后的卡顿与闪退风险
- 压缩预处理失败时输出明确日志并自动回退使用原图继续导出

### 图片排版导出 — 新增行内排列方向选项

- 排版布局区新增「行内排列方向」下拉框，支持三种模式：
  - **从左到右**（默认）：图片紧靠左侧排列，与原有行为一致
  - **居中对齐**：每行图片在可用宽度内水平居中排列
  - **从右到左**：图片整体靠右对齐（图片顺序仍从左到右）
- 末行图片数不满时，居中/右对齐效果尤为明显
- 该设置可保存到预设并恢复

### 打包配置 — 修复 exe 运行时 `No module named 'email'` 错误

- 从 PyInstaller `excludes` 列表中移除 `email` 模块，并在 `hiddenimports` 中显式补充 `email`、`email.mime.text`、`email.mime.multipart`
- 根因：`python-pptx` 内部依赖 `email` 模块解析 OOXML 包结构，原配置将其排除导致图片排版导出添加固定文本叠加层时崩溃

## 2026-04-27

### 主窗口 — Tab 标签栏合并优化

- **去除冗余双层标题**：移除顶部独立的 Tab 按钮行（图像处理/后台日志/版本日志），将 Tab 标签直接整合到内容区顶部，与内容卡片无缝衔接
- **节省垂直空间**：消除了原有"顶部Tab栏 + 内容区GroupBox标题"的双重标题冗余，右侧内容区可用高度明显增加



### 图片排版导出 & 图片叠加 — 元素列表 UI 优化

- **左右分栏布局**：元素列表区域改为左侧列表 + 右侧配置面板的左右分栏结构，右侧配置区获得更充足的水平空间
- **图标按钮**：添加文字/图片、删除、上移、下移操作按钮改为 emoji 图标样式（📝 🖼 🗑 ⬆ ⬇），视觉更简洁
- **重命名功能**：新增 ✏ 重命名按钮，可自定义元素列表中每个元素 `|` 后面的名称，方便区分多个同类元素
- **名称持久化**：自定义元素名称随预设一起保存和恢复
- **新增 `config.ICON_BTN_STYLE`**：统一的 emoji 图标按钮样式常量，供各模块复用

### 参数面板 — 切换功能模块自动回顶

- **滚动条自动重置**：切换功能模块时，右侧参数面板的滚动条自动回到顶部，无需手动滚动即可操作新模块的参数

### 图片排版导出 — 进度与日志优化

- **修复进度条显示错误**：排版导出输出单个文件（如 1 个 PPT）时，进度条现在正确显示已处理的图片数量（如 5/5），而非输出文件数（1/5）
- **增加逐页处理日志**：导出过程中实时输出每页的处理进度，格式为「第 N/M 页  [第x~y张]  文件名」，方便查看处理过程
- **优化完成日志**：处理完成后日志显示完整摘要，包含总图片数、总页数和输出文件名（如「✓ 分组: 排版导出  共 5 张图 → 5 页  →  排版导出.pptx」）
- **进度条动态同步**：进度条最大值与实际文件总数动态同步，避免批量合并处理器与逐张处理器之间的进度单位不一致问题
- **支持三种格式**：上述优化同时适用于 PPTX、PDF、DOCX 三种导出格式
